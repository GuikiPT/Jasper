# Commands Catalogue

This document provides comprehensive reference for all Jasper Revamp commands, including permissions, options, and usage patterns.

## Documentation Conventions

- **Role Buckets**: Permission categories configured via `/settings roles`
- **Command Syntax**: Both slash commands (`/command`) and prefix commands (`j!command`) are supported
- **Default Prefix**: `j!` (configurable per guild)
- **Ephemeral**: Response visible only to the command invoker

## Permission Roles Reference

Commands use role-based access control through these permission buckets:

| Bucket | Purpose |
| --- | --- |
| **Support Roles** | Basic support utilities and tag access |
| **Allowed Tag Roles** | Permission to send and manage tags |
| **Allowed Tag Admin Roles** | Advanced tag administration (import/export/raw) |
| **Allowed Staff Roles** | Day-to-day moderation tools |
| **Allowed Admin Roles** | Full configuration access |
| **Ignored Sniped Roles** | Excluded from `/snipe` results |

Configure these via `/settings roles add` and `/settings roles remove`.

## General Commands

<a id="help"></a>
### `/help` - Interactive Command Reference

Provides searchable command catalogue with autocomplete. Returns category overview or detailed command information.

**Availability**: Guilds, Direct messages  
**Permissions**: Support Roles, Tag Roles, Staff Roles, Admin Roles  
**Options**: `command` (string, optional, autocomplete) - Command path like `settings roles add`

**Syntax**: `/help [command]` | `j!help [command]`

<a id="ping"></a>
### `/ping` - Latency Measurement

Measures Discord gateway (websocket heartbeat) and interaction round-trip latency.

**Availability**: Guilds, Direct messages  
**Permissions**: None  
**Aliases**: `latency` (message command)

**Syntax**: `/ping` | `j!ping` | `j!latency`

<a id="topic"></a>
### `/topic` - Random Discussion Topic

Sends a random discussion topic from the guild's configured topic pool.

**Availability**: Guild channels  
**Permissions**: Staff Roles, Admin Roles, Manage Server

**Syntax**: `/topic` | `j!topic`

**Note**: Topics configured via `/settings topics`

## Moderation Commands

<a id="snipe"></a>
### `/snipe` - Recover Deleted Message

Retrieves the most recently deleted message in the current channel. Command reply is ephemeral; recovered content posts publicly.

**Availability**: Allow-listed guild channels  
**Permissions**: Staff Roles, Admin Roles

**Syntax**: `/snipe` | `j!snipe`

**Requirements**: 
- Channel must be in `allowedSnipeChannels` (configure via `/settings channels add`)
- Messages from `ignoredSnipedRoles` are excluded from recovery

<a id="virustotal"></a>
### `/virustotal` - Security Analysis

Submits files, URLs, domains, and IP addresses to VirusTotal for comprehensive threat analysis.

**Availability**: Guild channels  
**Permissions**: Staff Roles, Admin Roles, Manage Server  
**Common Option**: `ephemeral` (boolean, optional) - Makes response visible only to you (default: true)

**Subcommands**:
- [`ip`](#virustotal-ip) - Analyze IP address reputation and malicious activity
- [`domain`](#virustotal-domain) - Check domain reputation and security status
- [`file`](#virustotal-file) - Upload and scan files for malware (max 650MB)
- [`url`](#virustotal-url) - Scan URLs for malicious content

All results include detection ratios from multiple antivirus engines, detailed reports as downloadable files, and links to web-based VirusTotal reports.

<a id="virustotal-ip"></a>
#### `/virustotal ip` - IP Address Analysis

Analyzes IP address for malicious activity and reputation.

**Options**:
- `address` (string, required) - IP address to analyze
- `ephemeral` (boolean, optional) - Response visibility

**Response Includes**: Security status, geographic info (country, AS owner), detection summary, community votes, reputation score, downloadable report

**Syntax**: `/virustotal ip address:<ip> [ephemeral:true|false]`

<a id="virustotal-domain"></a>
#### `/virustotal domain` - Domain Analysis

Checks domain reputation and malicious activity across security sources.

**Options**:
- `name` (string, required) - Domain name to analyze
- `ephemeral` (boolean, optional) - Response visibility

**Response Includes**: Security assessment, domain categories, detection summary, community votes, threat intelligence, downloadable report

**Syntax**: `/virustotal domain name:<domain> [ephemeral:true|false]`

<a id="virustotal-file"></a>
#### `/virustotal file` - File Malware Scan

Uploads and scans files for malware using comprehensive analysis from 70+ antivirus engines.

**Options**:
- `attachment` (file, required) - File to scan (max 650MB)
- `ephemeral` (boolean, optional) - Response visibility

**Response Includes**: Threat assessment, file hashes (SHA256, MD5, SHA1), size/type info, malicious detections by engine, progress tracking, downloadable report

**Security Features**: Secure download with integrity verification, memory monitoring, hash verification, rate limiting, automatic retry with exponential backoff

**Syntax**: `/virustotal file attachment:<file> [ephemeral:true|false]`

<a id="virustotal-url"></a>
#### `/virustotal url` - URL Threat Scan

Submits URLs to VirusTotal for security scanning and threat analysis.

**Options**:
- `link` (string, required) - URL to scan
- `ephemeral` (boolean, optional) - Response visibility

**Response Includes**: Security status, URL info (title, HTTP status, reputation), detection summary, community votes, categories, progress tracking, downloadable report

**Syntax**: `/virustotal url link:<url> [ephemeral:true|false]`

## Support Commands

<a id="resolve"></a>
### `/resolve` - Close Support Thread

Applies configured resolved tag, optionally posts Q&A summary, then locks and archives the support thread.

**Availability**: Threads in configured support forum channel  
**Permissions**: Tag Roles, Staff Roles, Admin Roles  
**Bot Requirements**: Manage Threads permission in forum

**Options**:
- `question` (string, optional, max 1000 chars) - Summary of original request
- `answer` (string, optional, max 2000 chars) - Resolution notes

**Syntax**: `/resolve [question] [answer]`

**Auto-Resolution**: Jasper monitors support threads for inactivity. After the configured reminder delay, it prompts the thread author to keep open or close. If ignored, threads auto-resolve after the grace period. Configure via `/settings support set`.

<a id="tag"></a>
### `/tag` - Support Tag Management

Manages reusable support tags for quick responses.

**Availability**: Guilds only  
**Base Permissions**: Support Roles, Staff Roles, Admin Roles  
**Special Permissions**: Some subcommands require Tag Admin Roles  
**Channel Restrictions**: Sending tags may be limited to `allowedTagChannels`

**Subcommands**:
- [`create`](#tag-create) - Interactive workflow to add new tag
- [`delete`](#tag-delete) - Remove tag by name
- [`edit`](#tag-edit) - Update tag content
- [`export`](#tag-export) - Download all tags as JSON (Tag Admin only)
- [`import`](#tag-import) - Upload tags from JSON (Tag Admin only)
- [`info`](#tag-info) - Show tag metadata and usage stats
- [`list`](#tag-list) - List all available tags
- [`raw`](#tag-raw) - Display stored JSON payload (Tag Admin only)
- [`show`](#tag-show) - Preview tag without sending publicly
- [`use`](#tag-use) - Send tag to channel or user

<a id="tag-create"></a>
#### `/tag create` - Create New Tag

Starts interactive workflow to create a new support tag.

**Syntax**: `/tag create`

<a id="tag-delete"></a>
#### `/tag delete` - Delete Tag

Removes an existing tag by name.

**Options**: `name` (string, required, autocomplete) - Tag to delete

**Syntax**: `/tag delete name:<tag>`

<a id="tag-edit"></a>
#### `/tag edit` - Edit Tag

Updates the content of an existing tag.

**Options**: `name` (string, required, autocomplete) - Tag to update

**Syntax**: `/tag edit name:<tag>`

<a id="tag-export"></a>
#### `/tag export` - Export Tags

Downloads all tags as JSON file.

**Permissions**: Tag Admin Roles only

**Syntax**: `/tag export`

<a id="tag-import"></a>
#### `/tag import` - Import Tags

Imports tags from JSON text or uploaded file.

**Permissions**: Tag Admin Roles only

**Options**:
- `payload` (string, optional) - Raw JSON array inline
- `file` (attachment, optional) - Uploaded JSON file
- `overwrite` (boolean, optional) - Replace existing tags (true) or append (false)

**Syntax**: `/tag import [payload] [file] [overwrite]`

<a id="tag-info"></a>
#### `/tag info` - Tag Metadata

Shows metadata including creator and usage statistics.

**Options**: `name` (string, required) - Tag to inspect

**Syntax**: `/tag info name:<tag>`

<a id="tag-list"></a>
#### `/tag list` - List All Tags

Displays all available tags.

**Syntax**: `/tag list`

<a id="tag-raw"></a>
#### `/tag raw` - View Tag JSON

Displays the stored JSON payload for a tag.

**Permissions**: Tag Admin Roles only

**Options**: `name` (string, required) - Tag to inspect

**Syntax**: `/tag raw name:<tag>`

<a id="tag-show"></a>
#### `/tag show` - Preview Tag

Previews tag's rendered content without sending publicly.

**Options**:
- `name` (string, required) - Tag to preview
- `ephemeral` (boolean, optional) - Send preview ephemerally

**Syntax**: `/tag show name:<tag> [ephemeral:true|false]`

<a id="tag-use"></a>
#### `/tag use` - Send Tag

Sends saved tag to current channel or mentions user directly.

**Options**:
- `name` (string, required, autocomplete) - Tag to send
- `user` (user, optional) - Deliver via direct message to selected user

**Syntax**: `/tag use name:<tag> [user:<member>]`

**Note**: Public tag posting requires channel in `allowedTagChannels`

## Settings Commands

The `/settings` command configures guild-specific behavior. Users with "Manage Server" Discord permission can execute most settings actions alongside configured role buckets.

### Prefixes - Message Command Prefix

<a id="settings-prefixes-set"></a>
#### `/settings prefixes set` - Set Prefix

Sets message-command prefix for the server (max 16 characters).

**Permissions**: Admin Roles, Administrator

**Options**: `value` (string, required) - New prefix

**Syntax**: `/settings prefixes set value:<prefix>` | `j!settings prefixes set <prefix>`

<a id="settings-prefixes-view"></a>
#### `/settings prefixes view` - View Prefix

Shows current message-command prefix (or Jasper default if unset).

**Permissions**: Admin Roles, Administrator

**Syntax**: `/settings prefixes view` | `j!settings prefixes view`

### Roles - Permission Bucket Management

Manages role assignments to permission buckets controlling command access.

**Available Buckets**:
- `allowedAdminRoles` - Elevated administrators with full settings access
- `allowedStaffRoles` - Moderators for day-to-day tools
- `allowedTagAdminRoles` - Tag system administrators
- `allowedTagRoles` - Members who can send tags
- `ignoredSnipedRoles` - Roles excluded from `/snipe`
- `supportRoles` - Support utility access

<a id="settings-roles-add"></a>
#### `/settings roles add` - Add Role to Bucket

**Permissions**: Admin Roles, Tag Admin Roles, Manage Server

**Options**:
- `setting` (choice, required) - Permission bucket name
- `role` (role, required) - Role to add

**Syntax**: `/settings roles add setting:<bucket> role:<@role>` | `j!settings roles add <bucket> <@role>`

<a id="settings-roles-remove"></a>
#### `/settings roles remove` - Remove Role from Bucket

**Permissions**: Admin Roles, Tag Admin Roles, Manage Server

**Options**:
- `setting` (choice, required) - Permission bucket name
- `role` (role, required) - Role to remove

**Syntax**: `/settings roles remove setting:<bucket> role:<@role>` | `j!settings roles remove <bucket> <@role>`

<a id="settings-roles-list"></a>
#### `/settings roles list` - List Bucket Roles

Displays roles in one bucket or all buckets.

**Permissions**: Admin Roles, Tag Admin Roles, Manage Server

**Options**: `setting` (choice, optional) - Specific bucket name (omit to view all)

**Syntax**: `/settings roles list [setting]` | `j!settings roles list [bucket]`

### Channels - Feature Allow-Lists

Controls channel allow-lists for specific features.

**Available Buckets**:
- `allowedSnipeChannels` - Enable `/snipe` access
- `allowedTagChannels` - Enable sending saved tags
- `automaticSlowmodeChannels` - Opt into automatic slowmode

<a id="settings-channels-add"></a>
#### `/settings channels add` - Add Channel to Bucket

**Permissions**: Admin Roles, Administrator

**Options**:
- `setting` (choice, required) - Channel bucket name
- `channel` (channel, required) - Channel to register

**Syntax**: `/settings channels add setting:<bucket> channel:<#channel>` | `j!settings channels add <bucket> <#channel>`

<a id="settings-channels-remove"></a>
#### `/settings channels remove` - Remove Channel from Bucket

**Permissions**: Admin Roles, Administrator

**Options**:
- `setting` (choice, required) - Channel bucket name
- `channel` (channel, required) - Channel to remove

**Syntax**: `/settings channels remove setting:<bucket> channel:<#channel>` | `j!settings channels remove <bucket> <#channel>`

<a id="settings-channels-list"></a>
#### `/settings channels list` - List Bucket Channels

Lists channels in one bucket or all buckets.

**Permissions**: Admin Roles, Administrator

**Options**: `setting` (choice, optional) - Specific bucket (omit to view all)

**Syntax**: `/settings channels list [setting]` | `j!settings channels list [bucket]`

### Topics - Discussion Topic Pool

Maintains the pool of discussion topics used by `/topic` command.

<a id="settings-topics-add"></a>
#### `/settings topics add` - Add Topic

Adds a discussion topic to the guild pool. Duplicates are ignored.

**Permissions**: Admin Roles, Administrator

**Options**: `value` (string, required, max 256 chars) - Topic text

**Syntax**: `/settings topics add value:<text>` | `j!settings topics add <text>`

<a id="settings-topics-remove"></a>
#### `/settings topics remove` - Remove Topic

Removes topic via dropdown selection, list position, or exact text.

**Permissions**: Admin Roles, Administrator

**Options**:
- `topic` (string, optional, autocomplete) - Select from dropdown
- `position` (integer, optional) - Position number from list (1, 2, 3...)

**Syntax**: `/settings topics remove [topic:<text>] [position:<number>]` | `j!settings topics remove <topic_or_position>`

**Note**: Use either dropdown OR position, not both

<a id="settings-topics-list"></a>
#### `/settings topics list` - List Topics

Displays all configured topics with numeric IDs.

**Permissions**: Admin Roles, Administrator

**Syntax**: `/settings topics list` | `j!settings topics list`

<a id="settings-topics-import"></a>
#### `/settings topics import` - Import Topics

Imports topics from JSON text or attachment.

**Permissions**: Admin Roles, Administrator

**Options**:
- `payload` (string, optional) - Raw JSON array inline
- `file` (attachment, optional) - Uploaded JSON file
- `overwrite` (boolean, optional) - Replace existing (true) or append (false)

**Syntax**: `/settings topics import [payload] [file] [overwrite]` | `j!settings topics import [file] [text]`

<a id="settings-topics-export"></a>
#### `/settings topics export` - Export Topics

Exports all saved topics as JSON file.

**Permissions**: Admin Roles, Administrator

**Syntax**: `/settings topics export` | `j!settings topics export`

### Support - Forum Configuration

Configures support forum integration for `/resolve` command and automatic thread monitoring.

<a id="settings-support-set"></a>
#### `/settings support set` - Configure Support Settings

Sets support forum channel, resolved tag, and inactivity timers.

**Permissions**: Admin Roles, Tag Admin Roles, Manage Server

**Options**:
- `setting` (choice, required) - One of: `supportForumChannelId`, `resolvedTagId`, `inactivityReminderMinutes`, `autoCloseMinutes`
- `value` (string, optional) - Channel ID, Tag ID, or minutes (1-10080). Leave blank to clear channel/tag settings

**Syntax**: `/settings support set setting:<option> [value:<id>]` | `j!settings support set <option> [value]`

<a id="settings-support-view"></a>
#### `/settings support view` - View Support Configuration

Displays configured support forum, resolved tag, and inactivity timers.

**Permissions**: Admin Roles, Tag Admin Roles, Manage Server

**Syntax**: `/settings support view` | `j!settings support view`

### Slowmode - Automatic Slowmode

Controls automatic slowmode system that adjusts channel slowmode based on activity.

<a id="settings-slowmode-view"></a>
#### `/settings slowmode view` - View Slowmode Config

Shows automatic slowmode configuration and tracked channels.

**Permissions**: Admin Roles

**Syntax**: `/settings slowmode view` | `j!settings slowmode view`

<a id="settings-slowmode-configure"></a>
#### `/settings slowmode configure` - Configure Slowmode

Updates automation thresholds and timers for automatic slowmode.

**Permissions**: Admin Roles

**Options**:
- `enabled` (boolean, optional) - Enable/disable automatic slowmode
- `threshold` (integer, optional) - Messages required within window
- `window` (integer, optional) - Activity window in seconds
- `cooldown` (integer, optional) - Seconds between adjustments
- `reset` (integer, optional) - Inactivity seconds before timer resets to 1
- `max` (integer, optional) - Maximum slowmode duration (up to 21,600 seconds)

**Syntax**: `/settings slowmode configure [enabled] [threshold] [window] [cooldown] [reset] [max]` | `j!settings slowmode configure enabled=true threshold=10 window=60`

### YouTube - Subscriber Count Tracking

Configures automated YouTube subscriber tracking that updates a Discord channel name with live subscriber counts.

<a id="settings-youtube-enable"></a>
#### `/settings youtube enable` - Enable Tracking

Configures Jasper to poll a YouTube channel and update Discord channel name with subscriber count.

**Permissions**: Admin Roles, Manage Channels (required for invoker and bot)

**Options**:
- `youtube_url` (string, required) - YouTube channel URL (e.g., `https://www.youtube.com/@ChannelName`)
- `discord_channel` (channel, required) - Channel to rename with subscriber count
- `interval` (integer, optional) - Minutes between updates (5-1440, default: 30)

**Syntax**: `/settings youtube enable youtube_url:<url> discord_channel:<#channel> [interval:<minutes>]` | `j!settings youtube enable <youtube_url> <#channel> [interval_minutes]`

**Note**: Voice channels recommended for cleanest display

<a id="settings-youtube-disable"></a>
#### `/settings youtube disable` - Disable Tracking

Stops automatic subscriber tracking. Channel keeps current name.

**Permissions**: Admin Roles, Manage Channels

**Syntax**: `/settings youtube disable` | `j!settings youtube disable`

<a id="settings-youtube-view"></a>
#### `/settings youtube view` - View Configuration

Shows YouTube tracking configuration including status, channels, interval, and latest subscriber count.

**Permissions**: Admin Roles, Manage Channels

**Syntax**: `/settings youtube view` | `j!settings youtube view`

<a id="settings-youtube-test"></a>
#### `/settings youtube test` - Test Configuration

Validates configuration by forcing update and checking permissions.

**Permissions**: Admin Roles, Manage Channels

**Syntax**: `/settings youtube test` | `j!settings youtube test`

<a id="settings-youtube-force-update"></a>
#### `/settings youtube force-update` - Force Update

Immediately refreshes subscriber count outside normal interval.

**Permissions**: Admin Roles, Manage Channels

**Syntax**: `/settings youtube force-update` | `j!settings youtube force-update`

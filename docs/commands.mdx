# Commands Catalogue

This page documents every Jasper Revamp command with concise, authoritative details taken from the bot codebase. It focuses on what each command does, where it works, what permissions are needed, and which options it accepts.

Notes

- Examples and visual message previews have been intentionally removed.
- "Allowed" role buckets refer to categories configured via `/settings roles`.
- Message (prefix) commands use your configured prefix (default: `j!`).

## General Commands

### Quick Reference

| Command | Summary |
| --- | --- |
| [`/help`](#help) | Interactive, searchable command catalogue with autocomplete. |
| [`/ping`](#ping) | Measures gateway and interaction latency for Jasper. |

<a id="help"></a>
### `/help`

<details>
<summary>View details</summary>

**Purpose**

Provide an interactive command catalogue with autocomplete for commands and subcommands. Returns either a category overview or a detailed entry and links to the relevant documentation when available.

**Where it works**

- Guilds
- Direct messages

**Permissions**

- Support Roles
- Allowed Tag Roles
- Allowed Staff Roles
- Allowed Admin Roles

**Options**

- `command` (string, optional, autocomplete): Command or subcommand path such as `settings roles add`.
- `ephemeral` (boolean, optional): Show the response only to you. Defaults to `false` (public reply).

**Syntax**

`/help [command] [ephemeral:true|false]` | `j!help [command]`

</details>

<a id="automod-check"></a>
### `/automod-check`

<details>
<summary>View details</summary>

**Purpose**

Test words or phrases against the bot's automod rules to determine if they would be flagged or allowed.

**Where it works**

- Guild channels

**Permissions**

- Allowed Staff Roles
- Allowed Admin Roles

**Options**

- `content` (string, required): The word or phrase to check against automod rules (max 1000 characters).
- `ephemeral` (boolean, optional): Whether the response should be visible only to you. Defaults to `true`.

**Syntax**

`/automod-check content:"text to check" [ephemeral:true|false]`

**Examples**

```
/automod-check content:"discord.gg/test"
/automod-check content:"those clankers bro" ephemeral:false
/automod-check content:"hey <@1152813525912264834>"
/automod-check content:"check out https://cdn.discordapp.com/emojis/123.png"
/automod-check content:"I love custard pie"
```

**Response Format**

The command displays results in two color-coded containers:
- **Container 1**: Shows the checked content (neutral color)
- **Container 2**: Shows the result with color coding (red for blocked, green for clean/allowed)

**Response Types**

- ðŸš« **BLOCKED**: Content would be flagged by automod with details about which rule and pattern matched
- âœ… **ALLOWED**: Content is explicitly allowed despite matching a pattern, with exception details
- âœ… **CLEAN**: Content passes all automod checks with no violations detected

**Rule Categories**

The system checks against 5 comprehensive rule categories:

1. **Spoiler spam, markdown, soundmojis, staff ping** (63 blocked words, 10 regex patterns, 2 allowed exceptions)
   - Staff member pings and mentions
   - Special Unicode characters and formatting abuse
   - Soundmoji patterns and markdown manipulation
   - IP addresses and phone number links

2. **Soft Custom Keywords (with Links)** (114 blocked words, 3 regex patterns, 39 allowed exceptions)
   - Malicious domains and URL shorteners
   - Inappropriate content and mild profanity
   - Discord invite links (with NTTS server exceptions)
   - Scam and phishing patterns

3. **Medium Custom Keywords** (106 blocked words, 1 regex pattern, 45 allowed exceptions)
   - Sexual content and adult terminology
   - Moderate offensive language
   - Adult websites and services
   - Phone number detection patterns

4. **Hard Custom Keywords** (99 blocked words, 1 regex pattern, 18 allowed exceptions)
   - Racial slurs and hate speech
   - Severe offensive language
   - Advanced slur detection patterns
   - Nazi and discriminatory symbols

5. **Scam Nuker** (21 blocked words, 8 regex patterns, 1 allowed exception)
   - Known scam domains and services
   - Fake giveaway and nitro drop links
   - Malicious Steam community links
   - Complex scam link detection patterns

**Match Types**

- **Word/Phrase**: Direct text matches with wildcard support (`*`)
- **Regex Pattern**: Advanced pattern matching for complex rules
- **Special Character Handling**: Proper detection of Discord mentions, URLs, and special characters

**Notes**

- Rules are processed in order (1-5) and the first match determines the result
- Allowed words take precedence over blocked words within the same rule
- Uses word boundaries for normal text and contains matching for special characters
- Does not check for bypasses or obfuscation attempts
- Results display rule ID, name, match type, and specific pattern that triggered

**Technical Details**

- **Total Coverage**: 403 blocked words, 23 regex patterns, 105 allowed exceptions
- **Pattern Matching**: Uses word boundaries for normal text, contains matching for Discord mentions and URLs
- **Rule Processing**: Sequential evaluation (Rule 1â†’5), first match wins
- **Wildcard Support**: `*` wildcards supported in blocked/allowed words (e.g., `clanker*` matches `clankers`, `clankering`)
- **Exception Handling**: Allowed words override blocked words within the same rule
- **Performance**: Optimized for real-time checking with minimal false positives

**Use Cases**

- **Content Moderation**: Test if user messages would trigger automod
- **Rule Validation**: Verify new rules work as expected before deployment
- **Exception Testing**: Confirm allowed words bypass blocks correctly
- **Staff Training**: Help moderators understand what content gets flagged
- **Policy Development**: Analyze rule effectiveness and coverage gaps

</details>

<a id="virustotal"></a>
### `/virustotal`

<details>
<summary>View details</summary>

**Purpose**

Submit suspicious files, URLs, domains, and IP addresses to VirusTotal for comprehensive security analysis.

**Where it works**

- Guild channels

**Permissions**

- Allowed Staff Roles
- Allowed Admin Roles
- Support Roles
- Manage Server (Discord permission, implicit)

**Subcommands**

- [`ip`](#virustotal-ip) - Analyze an IP address for malicious activity.
- [`domain`](#virustotal-domain) - Check a domain for malicious activity and reputation.
- [`file`](#virustotal-file) - Upload and scan a file for malware.
- [`url`](#virustotal-url) - Scan a URL for malicious content.

**General Options (all subcommands)**

- `ephemeral` (boolean, optional): Whether the response should be visible only to you. Defaults to `true`.

**Notes**

- Results include detection ratios from multiple antivirus engines.
- All analysis data is provided by VirusTotal's comprehensive threat intelligence platform.
- Files up to 650MB are supported for scanning.
- Responses include detailed reports as downloadable text files.
- All `/virustotal` replies are ephemeral by default; set `ephemeral:false` to make them public.
- The bot reuses existing VirusTotal scans when a file hash already exists, and URL scans wait ~30s for analysis completion while respecting public rate limits.

</details>

<a id="resolve"></a>
### `/resolve`

<details>
<summary>View details</summary>

**Purpose**

Apply the configured resolved tag, optionally post a question and answer summary, then lock and archive the support thread.

> Jasper also monitors support threads for inactivity. After the configured reminder delay, the bot prompts the thread author to keep it open or close it; if ignored, the thread is resolved automatically after the grace period.

**Where it works**

- Threads inside the configured support forum channel

**Permissions**

- Allowed Tag Roles
- Allowed Staff Roles
- Allowed Admin Roles
- Jasper must have the Manage Threads permission in the forum

**Options**

- `question` (string, optional, up to 1000 characters): Summary of the original request.
- `answer` (string, optional, up to 2000 characters): Notes describing the resolution.

**Syntax**

`/resolve [question] [answer]`

</details>

### YouTube (`/settings youtube`)

Configures automated YouTube subscriber tracking that renames a Discord channel with the live subscriber count.

**Subcommands**
- [`/settings youtube enable`](#settings-youtube-enable) - Enable tracking for a YouTube channel and target Discord channel.
- [`/settings youtube disable`](#settings-youtube-disable) - Turn off subscriber tracking while keeping existing channel names untouched.
- [`/settings youtube view`](#settings-youtube-view) - Display the saved tracking configuration and the most recent metrics.
- [`/settings youtube test`](#settings-youtube-test) - Validate permissions and trigger an immediate update to confirm the setup.
- [`/settings youtube force-update`](#settings-youtube-force-update) - Manually force the subscriber count refresh outside the normal interval.

<a id="settings-youtube-enable"></a>
#### `/settings youtube enable`

<details>
<summary>View details</summary>

**Purpose**

Configure Jasper to poll a YouTube channel and update a Discord channel name with the latest subscriber count.

**Permissions**

- Allowed Admin Roles.
- Manage Channels (Discord permission, required for both the invoker and Jasper on the selected channel).

**Options**

- `youtube_url` (string, required): YouTube channel URL such as `https://www.youtube.com/@NoTextToSpeech`.
- `discord_channel` (channel, required): Text or voice channel whose name Jasper will keep in sync.
- `interval` (integer, optional): Minutes between updates (5-1440, defaults to 30).

**Syntax**

`/settings youtube enable youtube_url:<url> discord_channel:<#channel> [interval:<minutes>]` | `j!settings youtube enable <youtube_url> <#channel> [interval_minutes]`

**Additional Notes**

- Jasper validates both member and bot permissions before saving the configuration.
- Voice channels provide the cleanest display; text channels are supported but will also be renamed.

</details>

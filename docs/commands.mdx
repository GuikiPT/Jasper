# Commands Catalogue

This page documents every Jasper Revamp command with concise, authoritative details taken from the bot codebase. It focuses on what each command does, where it works, what permissions are needed, and which options it accepts.

Notes

- Examples and visual message previews have been intentionally removed.
- "Allowed" role buckets refer to categories configured via `/settings roles`.
- Message (prefix) commands use your configured prefix (default: `j!`).

## General Commands

### Quick Reference

| Command | Summary |
| --- | --- |
| [`/help`](#help) | Interactive, searchable command catalogue with autocomplete. |
| [`/ping`](#ping) | Measures gateway and interaction latency for Jasper. |

<details>
<summary>View details</summary>

<a id="resolve"></a>
### `/resolve`

<details>
<summary>View details</summary>

**Purpose**

Apply the configured resolved tag, optionally post a question and answer summary, then lock and archive the support thread.

> Jasper also monitors support threads for inactivity. After the configured reminder delay, the bot prompts the thread author to keep it open or close it; if ignored, the thread is resolved automatically after the grace period.

**Where it works**

- Threads inside the configured support forum channel

**Permissions**

- Allowed Tag Roles
- Allowed Staff Roles
- Allowed Admin Roles
- Jasper must have the Manage Threads permission in the forum

**Options**

- `question` (string, optional, up to 1000 characters): Summary of the original request.
- `answer` (string, optional, up to 2000 characters): Notes describing the resolution.

**Syntax**

`/resolve [question] [answer]`

</details>

## Settings Commands

The `/settings` command configures guild-specific behavior. Most actions are available as both slash and message commands. Where "Manage Server" is noted, users with that Discord permission may also execute the action.

### Quick Reference

| Category | Base Command | Subcommands |
| --- | --- | --- |
| Prefixes | `/settings prefixes` | `set`, `view` |
| Roles | `/settings roles` | `add`, `remove`, `list` |
| Channels | `/settings channels` | `add`, `remove`, `list` |
| Topics | `/settings topics` | `add`, `remove`, `list`, `import`, `export` |

| Slowmode | `/settings slowmode` | `view`, `configure` |
| YouTube | `/settings youtube` | `enable`, `disable`, `view`, `test`, `force-update` |

- [`/settings roles list`](#settings-roles-list) - View the roles assigned to buckets.

<a id="settings-roles-add"></a>
#### `/settings roles add`

<details>
<summary>View details</summary>

**Purpose**

Add a role to one of Jasper's permission buckets.

**Permissions**

- Allowed Admin Roles.
- Allowed Tag Admin Roles.
- Manage Server (Discord permission).

**Options**

- `setting` (choice, required): One of `allowedAdminRoles`, `allowedStaffRoles`, `allowedTagAdminRoles`, `allowedTagRoles`, `ignoredSnipedRoles`, `supportRoles`.
- `role` (role, required): Role to attach to the bucket.

**Syntax**

`/settings roles add setting:<bucket> role:<@role>` | `j!settings roles add <bucket> <@role>`

**Bucket choices**

- `allowedAdminRoles`: Elevated administrators who can change every Jasper setting.
- `allowedStaffRoles`: Moderators who can manage day-to-day tools.
- `allowedTagAdminRoles`: Specialists who manage saved tags.
- `allowedTagRoles`: Members permitted to send saved tags.
- `ignoredSnipedRoles`: Roles excluded from `/snipe` results.
- `supportRoles`: Roles recognized when using support utilities.

</details>

<details>
<summary>View details</summary>

Configure the support forum channel, resolved tag, and inactivity timers used by `/resolve` and the support thread monitor.

**Options**

`/settings support set setting:<option> [value:<id>]` | `j!settings support set <option> [value]`

**Validation**

`supportForumChannelId` must reference a forum channel. Minute-based options must be between 1 and 10080 (inclusive).

### YouTube (`/settings youtube`)

Configures automated YouTube subscriber tracking that renames a Discord channel with the live subscriber count.

**Subcommands**
- [`/settings youtube enable`](#settings-youtube-enable) - Enable tracking for a YouTube channel and target Discord channel.
- [`/settings youtube disable`](#settings-youtube-disable) - Turn off subscriber tracking while keeping existing channel names untouched.
- [`/settings youtube view`](#settings-youtube-view) - Display the saved tracking configuration and the most recent metrics.
- [`/settings youtube test`](#settings-youtube-test) - Validate permissions and trigger an immediate update to confirm the setup.
- [`/settings youtube force-update`](#settings-youtube-force-update) - Manually force the subscriber count refresh outside the normal interval.

<a id="settings-youtube-enable"></a>
#### `/settings youtube enable`

<details>
<summary>View details</summary>

**Purpose**

Configure Jasper to poll a YouTube channel and update a Discord channel name with the latest subscriber count.

**Permissions**

- Allowed Admin Roles.
- Manage Channels (Discord permission, required for both the invoker and Jasper on the selected channel).

**Options**

- `youtube_url` (string, required): YouTube channel URL such as `https://www.youtube.com/@NoTextToSpeech`.
- `discord_channel` (channel, required): Text or voice channel whose name Jasper will keep in sync.
- `interval` (integer, optional): Minutes between updates (5-1440, defaults to 30).

**Syntax**

`/settings youtube enable youtube_url:<url> discord_channel:<#channel> [interval:<minutes>]` | `j!settings youtube enable <youtube_url> <#channel> [interval_minutes]`

**Additional Notes**

- Jasper validates both member and bot permissions before saving the configuration.
- Voice channels provide the cleanest display; text channels are supported but will also be renamed.

</details>
